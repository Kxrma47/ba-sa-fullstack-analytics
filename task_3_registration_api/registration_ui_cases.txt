Задание 3. Интерпретация интерфейса регистрации
===============================================

Источник экрана
---------------

Предоставленный draw.io файл содержит интерфейс регистрации пользователя для страницы Book Store.

Видимые поля формы:

  First Name
  Last Name
  UserName
  Password
  CAPTCHA confirmation

Видимые кнопки:

  Register
  Back to Login

Успешный сценарий
-----------------

Пользователь заполняет First Name, Last Name, UserName, Password, подтверждает CAPTCHA и нажимает Register.

Сообщение об успехе на экране:

  User Register Successfully.

Ожидаемый результат backend:

  Пользователь создан.
  Пароль хранится только как hash.
  Username уникален.
  API возвращает 201 Created.

Ошибка: слабый пароль
---------------------

Сообщение на экране:

  Passwords must have at least one non alphanumeric character, one digit ('0'-'9'), one uppercase ('A'-'Z'), one lowercase ('a'-'z'), one special character and Password must be eight characters or longer.

Ожидаемый результат backend:

  API отклоняет запрос с клиентской ошибкой.
  API возвращает ошибку по полю password.
  Пользователь не создается.

Ошибка: пользователь уже существует
-----------------------------------

Сообщение на экране:

  User exists!

Ожидаемый результат backend:

  API проверяет уникальность username.
  API отклоняет повторный username.
  API возвращает 409 Conflict.
  Второй пользователь с тем же username не создается.

Ошибка: CAPTCHA не подтверждена
-------------------------------

Сообщение на экране:

  Please verify reCaptcha to register!

Ожидаемый результат backend:

  API требует CAPTCHA token от frontend.
  API проверяет token до создания пользователя.
  API возвращает 422 Unprocessable Entity, если CAPTCHA отсутствует или некорректна.
  Пользователь не создается.

Примечание по интерфейсу
------------------------

Предоставленный экран регистрации не содержит поле email. Поэтому REST API использует username как уникальный идентификатор входа. Валидация email не включена, потому что frontend не может отправить значение, которого нет на форме.
